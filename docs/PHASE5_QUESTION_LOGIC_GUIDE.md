## 设计原则

### triggerKeywords（触发关键词）
**作用**：当用户回答中包含这些关键词时，系统会优先选择 `followupQuestions` 中的题目进行深挖。

**选择原则**：
1. **具体性**：选择具体的人物、地点、职业、事件名词
   - ✅ 好：`["父亲", "母亲", "工厂", "参军", "下岗"]`
   - ❌ 差：`["很好", "不错", "一般"]`（太泛化）

2. **相关性**：关键词必须与后续追问直接相关
   - 例如：题目问"您在哪里工作？"
   - 关键词：`["工厂", "车间", "师傅", "同事"]`
   - 追问：`["work_005"]`（"您的师傅是个什么样的人？"）

3. **数量**：每题 3-8 个关键词

### followupQuestions（后续追问）
**作用**：当触发关键词命中时，系统会从这个列表中选择一个题目作为下一题。

**选择原则**：
1. **逻辑承接**：追问必须是当前题目的自然延伸
   - 例如：题目问"您的童年在哪里度过？"
   - 追问：`["childhood_003"]`（"您老家是什么样的？"）

2. **深度挖掘**：追问应该引导用户提供更多细节
   - 例如：题目问"您做什么工作？"
   - 追问：`["work_006", "work_007"]`（"您最难忘的一次工作经历？"、"您和同事关系怎么样？"）

3. **数量**：每题 1-3 个追问题目

## 输出格式要求

请按照以下 Markdown 表格格式输出：

```markdown
| 题目ID | triggerKeywords | followupQuestions | 说明 |
|--------|----------------|-------------------|------|
| childhood_001 | ["父母", "兄弟姐妹", "老家", "村子"] | ["childhood_003", "childhood_006"] | 如果提到家人，追问家庭成员和经济状况 |

```

**字段说明**：
- `triggerKeywords`：数组格式，3-8 个关键词
- `followupQuestions`：数组格式，1-3 个题目 ID
- `说明`：简要说明触发逻辑和追问意图

## 实施步骤

### 步骤 1：为现有 100 题添加逻辑关系字段
1. 阅读每个题目的内容和主题标签
2. 根据设计原则，为每题添加 `triggerKeywords`（3-8 个）
3. 根据逻辑承接关系，为每题添加 `followupQuestions`（1-3 个）
4. 输出 Markdown 表格

### 步骤 2：设计 50 个新题目
1. 按照上述分配建议，设计 50 个新题目
2. 每个新题目必须包含：
   - `id`（遵循命名规范，如 `childhood_016`）
   - `content`（题目内容）
   - `hint`（可选，辅助提示）
   - `options`（选项列表）
   - `topicTags`（主题标签）
   - `triggerKeywords`（触发关键词）
   - `followupQuestions`（后续追问）

### 步骤 3：验证逻辑关系
1. 检查每个 `followupQuestions` 中的题目 ID 是否存在
2. 检查逻辑关系是否合理（追问是否是当前题目的自然延伸）
3. 检查是否有循环引用（A → B → A）

## 注意事项

1. **避免循环引用**：A → B → C → A 这种循环会导致用户陷入死循环
2. **保持主题连贯**：追问题目应该与当前题目在同一主题或相邻主题
3. **关键词不要过于宽泛**："好"、"不错"、"一般"这类词不适合作为触发关键词
4. **追问不要过多**：每题最多 3 个追问，避免分支过多
5. **考虑老年人表达习惯**：关键词应该是老年人常用的口语词汇
