## 关键消费规则

### 1. Q1 逐人追问

`q01_entry` 选了 N 个人后，调度器按序逐人追问：
1. `q01_persona`（含占位符 `{PERSON}`，运行时替换为实际人名）
2. `q01_persona_age`
3. `q01_persona_saying`
4. `q01_persona_deed`

每追完一人，调度器检查是否还有剩余人选，给出口继续/跳过。所有人追完后自动跳转到 `q01_sibling_switch`。

### 2. 条件路由

| 条件 | 触发 |
|------|------|
| `has_children` | Q7 入口题 |
| `q02_entry` 选"独生" | 走 only 分支（q02_only_reason → q02_only_feeling） |
| `q06_entry` 选"伴侣" | 走 partner 链 |
| `q06_entry` 选"非伴侣" | 走 nonpartner 链 |

### 3. AI 动态追问

以下入口题的 L3+ 层由 `src/lib/question-ai.ts` 动态生成：
- Q1（逐人 deep 追问）
- Q3（童年记忆深挖）
- Q4（学校记忆深挖）
- Q5（职业细节 + 得意之事）
- Q6（伴侣初见 → 记忆 → 现状 → 遗憾 → 寄语）
- Q7（子女聚焦）
- Q10（成就/人物/灾难三分叉）

### 4. 自适应阈值

```
≤16 字  → deepen（素材不足，生成选项继续追问）
17-50字 → light_followup（已有素材，轻改写追问）
>50字   → close（素材充分，收束）
```
